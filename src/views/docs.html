API de Gerenciamento de Salas, Funcionários e Reservas

Esta aplicação expõe três conjuntos de endpoints REST em /api/employees, /api/rooms e /api/bookings. Todos aceitam e devolvem JSON.

1. Employees (Funcionários)

  • GET /api/employees – Lista todos os funcionários.
  • POST /api/employees – Cria um funcionário.
    Exemplo: {"nome":"Ana","departamento":"RH"}
  • PUT /api/employees/:id – Atualiza um funcionário.
  • DELETE /api/employees/:id – Remove um funcionário.

2. Rooms (Salas)

  • GET /api/rooms – Lista todas as salas.
  • POST /api/rooms – Cria uma sala.
    Exemplo: {"numero_sala":101,"capacidade_maxima":20}
  • PUT /api/rooms/:id – Atualiza uma sala.
  • DELETE /api/rooms/:id – Remove uma sala.

3. Bookings (Reservas)

  • GET /api/bookings – Lista todas as reservas.
  • POST /api/bookings – Cria uma reserva.
    Exemplo: {"employee_id":1,"room_id":2,"dia_hora_reserva":"2025-05-23T14:00:00","num_pessoas":5}
  • PUT /api/bookings/:id – Atualiza uma reserva.
  • DELETE /api/bookings/:id – Remove uma reserva.

Exemplos de uso via curl

curl -X POST http://localhost:3000/api/employees \
     -H "Content-Type: application/json" \
     -d '{"nome":"Ana","departamento":"RH"}'

curl -X POST http://localhost:3000/api/rooms \
     -H "Content-Type: application/json" \
     -d '{"numero_sala":101,"capacidade_maxima":20}'

curl -X POST http://localhost:3000/api/bookings \
     -H "Content-Type: application/json" \
     -d '{"employee_id":1,"room_id":2,"dia_hora_reserva":"2025-05-23T14:00:00","num_pessoas":5}'
